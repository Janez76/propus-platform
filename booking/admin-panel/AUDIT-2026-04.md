## Gefundene Probleme

### Blocker (❌)

**1. Hardcoded VAT-Konstante in useWizardForm.ts**
- **Datei**: `src/components/orders/CreateOrderWizard/useWizardForm.ts`
- **Zeile**: 298
- **Problem**: `const vat = Math.round(vatBase * 0.081 * 100) / 100;` – nutzt hardcodiert `0.081` statt der Konstante `VAT_RATE` aus `src/lib/pricing.ts`
- **Auswirkung**: Wenn VAT-Rate sich ändert, ist dieser Ort nicht konsistent mit `calculatePricing`. Violiert DRY-Prinzip.
- **Empfohlener Fix**: `import { VAT_RATE } from "../../../lib/pricing"` und ersetze `0.081` durch `VAT_RATE`

### Einschränkungen (⚠️)

**2. OrderDetail bleibt monolithisch (1329 Zeilen)**
- **Datei**: `src/components/orders/OrderDetail.tsx`
- **Problem**: Ursprünglich war Extraction zu `OrderDetail/` mit Subkomponenten (Header, StatsBar, Tabs, Inline-Hook) geplant. Stattdessen: Header + StatsBar separat, aber große Detail-Logik bleibt im Root.
- **Auswirkung**: Schwerer zu testen, wartbar, aber funktioniert. Für Phase 2c reserviert.
- **Status**: Akzeptiert (bewusste Entscheidung in Phase 2b: Priorität auf Per-Card-Edit UX statt Datei-Aufteilung)

**3. ESLint: 13 pre-bestehende Fehler / Warnungen**
- **Zusammenfassung**: Hauptsächlich `react-hooks/exhaustive-deps` (lang-Dependencies), `react-hooks/set-state-in-effect` (UploadTool), `react-hooks/static-components` (OrderTable SortHeader inside render)
- **Auswirkung**: Kein neuer Code ist betroffen, aber sollten irgendwann gefixt werden
- **Empfohlener Fix**: Separater Lint-Cleanup-PR, z.B. mit ESLint-Disable-Comments + Jira-Ticket

**4. Empty States nicht konsequent genutzt**
- **Datei**: `src/components/orders/OrderDetail.tsx` (Tabs)
- **Problem**: `src/components/ui/empty-state.tsx` existiert, wird aber nicht verwendet. Stattdessen `{list.length === 0 ? null : <...>}` inline.
- **Auswirkung**: Keine konsistente Empty-State-UX, aber Funktionalität OK
- **Empfohlener Fix**: Bei nächstem Tab-Refactor (Phase 2c) integrieren
